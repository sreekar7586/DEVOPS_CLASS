\# Day 27 - Microservices Architecture



Microservices architecture is a modern approach to building applications.



\## What is Microservices?



\- Application is divided into small independent services

\- Each service performs a specific function

\- Services communicate via APIs



\## Monolithic vs Microservices



\### Monolithic



\- Single large application

\- All components tightly coupled

\- Hard to scale



\### Microservices



\- Small independent services

\- Loosely coupled

\- Easy to scale and maintain



\## Example



E-commerce application:



\- User Service

\- Product Service

\- Order Service

\- Payment Service



Each runs separately.



\## Advantages



\- Scalability

\- Flexibility

\- Fault isolation

\- Faster deployment



\## Disadvantages



\- Complex management

\- Requires networking

\- Debugging is harder



\## Relation with Docker



\- Each microservice runs in a container

\- Docker helps isolate services

\- Docker Compose manages multiple services



\## Real World Usage



\- Netflix

\- Amazon

\- Flipkart



\## Summary



Microservices architecture improves scalability and flexibility using independent services.
