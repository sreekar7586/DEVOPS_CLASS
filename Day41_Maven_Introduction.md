\# Day 41 - Maven Introduction and Build Lifecycle



Maven is a build automation and dependency management tool.



\## Why Maven?



Before Maven:

\- Manual compilation

\- Dependency issues

\- Difficult project management



Maven automates these tasks.



\## Features of Maven



\- Build automation

\- Dependency management

\- Standard project structure

\- Plugin support



\## What is POM?



POM stands for Project Object Model.



File name:

pom.xml



Contains:

\- Project details

\- Dependencies

\- Plugins

\- Build configuration



\## Maven Directory Structure



src/main/java

src/test/java

pom.xml



\## Maven Build Lifecycle



1\. validate

2\. compile

3\. test

4\. package

5\. verify

6\. install

7\. deploy



\## Important Maven Commands



\### Compile Project



mvn compile



\### Run Tests



mvn test



\### Package Application



mvn package



\### Install to Local Repository



mvn install



\## Maven Dependencies



Dependencies are libraries required for application.



Example:

\- Spring Boot

\- JUnit

\- MySQL connector



\## Advantages



\- Faster builds

\- Automatic dependency download

\- Better project management



\## Notes



\- Maven uses XML configuration

\- Central repository stores dependencies

\- Widely used in Java projects



\## Summary



Maven simplifies building, testing, and managing Java applications efficiently.
