\# Day 34 - Docker Logs and Monitoring



Docker provides commands to monitor containers and troubleshoot issues.



\## 1. Docker Logs



docker logs <container\_id>



Displays logs generated by a container.



Example:



docker logs nginx-container



Useful for:

\- Checking errors

\- Viewing application output

\- Debugging failed containers



\## 2. Follow Logs in Real Time



docker logs -f nginx-container



\-f means follow mode.



Shows live logs continuously.



\## 3. Show Last Few Lines



docker logs --tail 20 nginx-container



Displays last 20 lines of logs.



\## 4. Docker Stats



docker stats



Shows live resource usage of containers.



Information includes:

\- CPU usage

\- Memory usage

\- Network usage

\- Container ID



\## 5. Docker Top



docker top <container\_id>



Shows running processes inside a container.



\## 6. Docker Events



docker events



Displays real-time Docker events.



Useful for monitoring:

\- Container start

\- Container stop

\- Image pull



\## 7. Practical Example



docker run -d --name mynginx nginx



docker logs mynginx



docker stats



docker top mynginx



\## Notes



\- Logs help identify problems quickly

\- Monitoring improves container management

\- Important in production environments



\## Common Troubleshooting



Container exited immediately:

\- Check docker logs



High memory usage:

\- Check docker stats



\## Summary



Docker monitoring commands help track container performance and troubleshoot issues efficiently.
